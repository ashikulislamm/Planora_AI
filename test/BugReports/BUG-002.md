# Bug Report: BUG-002

**Title**: Lack of loading skeleton causes layout shift on Dashboard load
**Environment**: iOS 16, Safari, Mobile (375px)
**Module**: Dashboard & Analytics (DASH)

## Steps to Reproduce
1. Set network speed to "Slow 3G" in DevTools (to simulate poor mobile connection).
2. Log into the application.
3. Observe the initial render of the `/dashboard` route.

## Expected Result
A skeleton loader or spinner should reserve the vertical space for the Overview cards and Activity feed to prevent Cumulative Layout Shift (CLS).

## Actual Result
The page renders mostly blank, and when the API response (`/analytics/overview`) returns, the cards suddenly pop into existence, pushing the rest of the content down abruptly.

## Severity
Medium (UX Issue)

## Priority
Medium

## Status
Open

## Recommendation
Implement `loading.tsx` for the dashboard segment in Next.js, utilizing skeleton UI components that match the dimensions of the final loaded cards.
