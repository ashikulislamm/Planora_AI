# Bug Report: BUG-004

**Title**: Rapidly clicking 'Add Subtask' creates duplicate empty entries
**Environment**: Windows 11, Edge, Desktop
**Module**: Task Management (TASK)

## Steps to Reproduce
1. Open the "My Tasks" page.
2. Click on a task to view details.
3. Click the "Add Subtask" button 5 times very rapidly in succession before the first network request finishes.

## Expected Result
The button should be disabled (`disabled={isLoading}`) or debounced while the API request (`POST /tasks/:id/subtasks`) is processing to prevent multiple submissions.

## Actual Result
The API receives 5 requests, inserting 5 identical blank subtasks into the task object.

## Severity
Medium

## Priority
Medium

## Status
Open

## Recommendation
Implement a loading state on the button that sets `disabled={true}` while the Axios/Fetch request is resolving. Alternatively, debounce the click event.
