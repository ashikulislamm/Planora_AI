# Bug Report: BUG-001

**Title**: Focus Space timer state is lost upon hard refresh (Ctrl+F5)
**Environment**: Windows 11, Google Chrome 114, Desktop
**Module**: Focus Space (FOC)

## Steps to Reproduce
1. Log into the application.
2. Navigate to "Focus Space" via the sidebar.
3. Click "Start Session" to begin a 25-minute Pomodoro.
4. Wait for the timer to reach 24:30.
5. Press Ctrl+F5 to hard refresh the page.

## Expected Result
The timer state should persist via localStorage or backend session, and upon reload, it should calculate elapsed time and resume from ~24:30.

## Actual Result
The timer completely resets back to the default 25:00 state, and the active session history is lost.

## Severity
High (Data loss / Workflow interruption)

## Priority
High

## Status
Open

## Recommendation
Store the `sessionStartTime` and `expectedEndTime` in `localStorage` or push the active session state to the database immediately upon clicking "Start". On component mount (`useEffect`), check for an active session and resume calculating time differences.
