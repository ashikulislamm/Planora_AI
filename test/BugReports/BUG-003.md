# Bug Report: BUG-003

**Title**: Task title does not wrap properly on mobile view causing horizontal scroll
**Environment**: Android 13, Chrome Mobile, 320px width
**Module**: Task Management (TASK)

## Steps to Reproduce
1. Log in.
2. Navigate to "My Tasks".
3. Create a task with an unbroken string title (e.g., "ReviewingTheEntireCodebaseWithoutAnySpacesOrBreaksForTestingPurposes").
4. View the task list on a device with 320px width.

## Expected Result
The text should truncate with an ellipsis (`text-overflow: ellipsis`) or break the word (`word-break: break-all`) to fit within the container.

## Actual Result
The unbroken text forces the container to expand, introducing an ugly horizontal scrollbar to the entire application body.

## Severity
Low (Cosmetic/UX)

## Priority
Medium

## Status
Open

## Recommendation
Add the Tailwind utility classes `truncate` or `break-words whitespace-normal` to the task title span element inside `TaskPreviewCard.tsx`.
